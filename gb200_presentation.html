§3 — Grace Hopper Superchip

GH200 — The Superchip

Not a CPU talking to a GPU. A unified processor with coherent memory across 900 GB/s.

Coherent Memory

Grace CPU can directly address the Hopper GPU's HBM3 memory — and vice versa — without explicit copy operations. The OS sees one unified memory space.

vs PCIe approach

Traditional: CPU computes → copy to GPU VRAM over PCIe → GPU processes. GH200: both dies share the same address space. Zero-copy transfers.

§4 — Blackwell Architecture

GB200 — Blackwell

One Grace CPU + two Blackwell GPU dies. The chip that hit the physical limit of what can be printed.

The Blackwell GPU is so large it hits the "Reticle Limit" — the physical maximum size that lithography machines can expose on a wafer. To go larger, NVIDIA connected two full-size dies with a 10 TB/s interconnect, making them appear as a single chip to software.

§4 — Blackwell Innovations

Architectural Innovations

Three breakthroughs that define Blackwell's dominance in HFT and AI inference.

HBM3e
Memory
8 TB/s of memory bandwidth — 2.4× faster than GDDR6X. HBM (High Bandwidth Memory) stacks DRAM dies vertically on the same package as the GPU, connected by thousands of Through-Silicon Vias (TSV). Critical for HFT: loading a 100 GB order book from memory takes 12.5 microseconds instead of milliseconds over PCIe.
Decompression
Engine
800 GB/s dedicated decompression hardware — handles LZ4 and Deflate in silicon, completely offloading the Grace CPU. Real-world financial data is often compressed 3–5×. Blackwell decompresses a 10 GB compressed dataset in 12.5 ms vs 500+ ms on CPU. This keeps the compute pipeline fed.
FP4 Precision
2nd Gen TE
4-bit floating point via the 2nd-Gen Transformer Engine. For AI inference, full FP32 precision is unnecessary — the model's accuracy barely changes with FP4. But throughput doubles: the same physical chip can run 2× larger models or process 2× more queries per second. Enables LLM inference at 120 TFLOPS per GPU.

§5 — Conclusion

The Architectural Journey

45 years of solving the same problem: getting data to compute, faster.

HFT Relevance

In HFT, latency is measured in microseconds. By moving data at 10 TB/s using unified memory and on-chip decompression, the GB200 eliminates the data-starvation bottleneck that has constrained quantitative trading strategies for a decade.

Key Insight

Every breakthrough was about closing the gap between compute speed and data movement speed. The 8086 had no gap. Today's systems have eliminated it entirely — by making memory and compute one unified architecture.
